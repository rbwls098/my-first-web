# 부록 B. AI 사용 로그 템플릿

> **목적**: Copilot을 사용하면서 발생한 실수와 해결 과정을 기록한다. 기말 프로젝트 필수 제출물이다.

---

## B.1 작성 방법

AI 사용 로그는 **AI가 틀렸을 때마다** 한 건씩 기록한다. 완벽하게 동작한 경우는 기록하지 않아도 된다.

**기록 시점**: 오류를 발견하고 해결한 직후에 바로 작성한다. 나중에 몰아서 쓰면 기억이 흐려진다.

---

## B.2 로그 양식

```text
───────────────────────────────────────────
[날짜]     YYYY-MM-DD
[챕터]     Ch00
[프롬프트] AI에게 요청한 내용을 그대로 적는다
[AI 실수]  AI가 생성한 코드 중 틀린 부분을 적는다
[분류]     버전 불일치 / 컨텍스트 소실 / 환각 중 하나
[해결]     어떻게 수정했는지 적는다
[조치]     copilot-instructions.md에 추가한 규칙 (없으면 "없음")
───────────────────────────────────────────
```

---

## B.3 작성 예시

### 예시 1 — 버전 불일치

```text
───────────────────────────────────────────
[날짜]     2026-03-10
[챕터]     Ch2
[프롬프트] Next.js App Router 프로젝트에서 현재 경로를 가져오는 훅을 추가해줘
[AI 실수]  import { useRouter } from 'next/router' 로 생성함
[분류]     버전 불일치
[해결]     import { usePathname } from 'next/navigation' 으로 변경
[조치]     copilot-instructions.md Known AI Mistakes에 추가:
           "next/router 사용 금지 → next/navigation 사용"
───────────────────────────────────────────
```

### 예시 2 — 환각

```text
───────────────────────────────────────────
[날짜]     2026-03-17
[챕터]     Ch5
[프롬프트] Supabase에서 여러 행을 한 번에 업데이트하는 코드를 작성해줘
[AI 실수]  supabase.from('posts').upsertMany([...]) 메서드를 사용함 — 존재하지 않는 API
[분류]     환각
[해결]     공식 문서 확인 후 supabase.from('posts').upsert([...]) 로 수정
[조치]     없음 (copilot-instructions.md에 Supabase 섹션 추후 추가 예정)
───────────────────────────────────────────
```

### 예시 3 — 컨텍스트 소실

```text
───────────────────────────────────────────
[날짜]     2026-03-24
[챕터]     Ch6
[프롬프트] 로그인 버튼을 헤더에 추가해줘
[AI 실수]  이미 layout.js에 있는 Header 컴포넌트를 무시하고
           page.js에 새로운 nav 태그를 생성함
[분류]     컨텍스트 소실
[해결]     @workspace를 붙여 재요청 → 기존 Header 컴포넌트에 올바르게 추가됨
[조치]     copilot-instructions.md에 추가:
           "헤더/네비게이션 수정 시 반드시 app/components/Header.js를 먼저 확인"
───────────────────────────────────────────
```

## B.4 분류 기준

| 분류 | 판단 기준 |
|------|----------|
| **버전 불일치** | AI가 제안한 API/문법이 현재 설치된 버전에서 변경되거나 삭제된 경우 |
| **컨텍스트 소실** | 프로젝트에 이미 있는 파일/컴포넌트를 AI가 모르고 중복 생성하거나, 엉뚱한 파일을 수정한 경우 |
| **환각** | AI가 제안한 패키지명, 함수명, 옵션이 실제로 존재하지 않는 경우 |

---

## B.5 로그 파일 관리

프로젝트 루트의 `AI_LOG.md` 파일에 기록한다. 파일에는 양식과 Ch2~Ch13 섹션이 미리 준비되어 있다.

해당 챕터 섹션 아래에 로그를 추가하면 된다:

```markdown
## Ch2

───────────────────────────────────────────
[날짜]     2026-03-10
[챕터]     Ch2
[프롬프트] ...
[AI 실수]  ...
[분류]     버전 불일치
[해결]     ...
[조치]     ...
───────────────────────────────────────────
```

챕터별로 구분되어 있어 최종 제출 시 정리하기 편하다.

---

## B.6 기말 제출 체크리스트

| 항목 | 기준 |
|------|------|
| 로그 건수 | 챕터당 최소 1건 이상 |
| 분류 | 3가지 분류 각각 최소 1건 이상 포함 |
| [조치] 항목 | 빈칸 없이 작성 ("없음"도 가능) |
| copilot-instructions.md | 로그의 [조치] 내용이 실제로 반영되어 있어야 함 |
